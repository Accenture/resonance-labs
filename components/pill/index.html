Pill

A pill is a small, rounded label used to tag, categorize, or indicate the status of an item — static variants convey information while interactive variants can be selected or removed.

Demo

Observe the static pills for their label and color meaning. If removable pills are present, try activating the dismiss control with a keyboard to confirm it is reachable and that removing a pill is announced to screen readers.

What to Observe

  • Static pills are rendered as non-interactive text labels and are not focusable.
  • Removable pills expose a dismiss control that is keyboard-accessible and has a descriptive accessible name.
  • The dismiss button's accessible name includes the pill's label so users know exactly what they are removing.
  • Color is not the only means of conveying the pill's category or status — text labels are always present.
  • When a pill is removed, focus moves to a logical next target rather than being dropped.

Anatomy

[Anatomy image placeholder — will be added when assets are available]

  1. Pill container: The rounded wrapper element that groups the label and optional dismiss control.
  2. Label text: The visible text inside the pill that identifies the tag, category, or status it represents.
  3. Status indicator: An optional colored dot or border that provides a secondary visual cue about the pill's meaning.
  4. Dismiss button: An interactive control, present only on removable pills, that deletes the pill when activated.
  5. Dismiss icon: A visual symbol (such as an x) on the dismiss button, supplemented by an accessible label.

Accessibility Behavior

  • Static pills must not be interactive — they should not be focusable or receive keyboard events.
  • Removable pills must expose their dismiss control as a button, reachable by Tab and activatable by keyboard.
  • The dismiss button must have an accessible name that includes the pill's label so users know what will be removed.
  • Status or category information conveyed by color must also be communicated through text or an accessible label.
  • When a pill is removed from the page, focus must move to a sensible adjacent element — not disappear entirely.
  • If pills represent selected filter states, the overall filter group should be perceivable as a related set.

Common Mistakes

  • Adding a dismiss button with only an icon and no accessible name, so screen reader users hear "button" with no context.
  • Making static display pills focusable or interactive when they carry no action.
  • Conveying the pill's meaning or status through color alone, excluding users who cannot perceive color differences.
  • Dropping focus to the top of the page or to an unrelated element after a pill is dismissed.
  • Using div or span elements for removable pills instead of a proper button for the dismiss control.

Why This Matters

Pills are widely used to surface applied filters, tags, and statuses inline within interfaces. When a dismiss button has no accessible name, a screen reader user cannot tell which item they are about to remove. When status is conveyed only through color, users with color vision deficiencies lose meaningful information. Small as they are, pills appear at high density — getting their accessibility right multiplies benefit across entire pages of tagged content.
