Accessible component patterns for enterprise teams

Resonance Labs provides accessible component patterns for real-world applications. Each component helps teams learn structure, behavior, and accessibility expectations. Content is designed for implementation learning and training.

How to use this site

  1. Select a component from the library below
  2. Explore the interactive demo
  3. Review the component anatomy
  4. Understand expected accessibility behavior
  5. Apply the pattern in your own implementation

Who this is for

  • Frontend developers building accessible interfaces
  • QA engineers validating component behavior
  • Designers understanding interaction and state requirements
  • Accessibility practitioners reviewing implementations
  • Team trainers and onboarding leads

What every component page covers

All components follow a consistent structure so teams can build a repeatable mental model:

  • Demo — interactive example to explore in the browser
  • Anatomy — labeled parts of the component
  • Accessibility Behavior — what must be true for the component to work for all users
  • Common Mistakes — implementation errors to watch for
  • Why It Matters — the impact of getting it wrong

Components are reviewed against internal accessibility standards aligned with WCAG.

Core

Accordion Expandable sections that show and hide content panels.
Alert Inline notification announced automatically via a live region.
Alert Dialog Modal dialog requiring confirm or cancel before continuing.
Autocomplete Text input with filtered suggestions as the user types.
Avatar User image or initials fallback with accessible labelling.
Badge Removable tag or filter chip with dismiss action.
Breadcrumbs Navigation trail showing the user's location in a hierarchy.
Button Clickable element that triggers an action or submits a form.
Card Contained surface for related content and actions.
Card List A collection of cards displayed as a navigable list.
Carousel Rotating set of slides with navigation controls.
Checkbox Toggle control for selecting one or more options.
Combobox Input with a dropdown list that supports typing and selection.
Cookie Consent Consent banner with accept, reject, and preference controls.
Date Picker Date input with calendar popup and keyboard navigation.
Dialog Non-modal dialog surface that does not block background content.
Disclosure Toggle widget that shows or hides a section of content.
Dropdown Menu that appears on trigger, presenting a list of actions.
File Upload File input with drag-and-drop, progress, and validation.
Link Inline navigation element that takes users to another page or section.
Listbox Single or multi-select list following the ARIA listbox pattern.
Mega Navigation Large multi-column dropdown for complex site navigation.
Modal Dialog overlay that requires user interaction before continuing.
Pagination Navigation control for paging through multi-page content.
Password Input Toggle Password field with a show/hide toggle for visibility.
Pill Compact label used to categorize or tag content.
Popover Non-modal floating panel toggled by a trigger button.
Progress Bar Visual indicator showing completion status of a task.
Radio Button Selection control for choosing one option from a group.
Search Input Text field designed for search queries with clear action.
Select Dropdown control for choosing a single value from a list.
Skip Navigation Hidden link that lets keyboard users bypass repetitive navigation.
Slider Range input with draggable thumb and keyboard support.
Table Structured data display organized in rows and columns.
Tabs Tabbed interface for switching between content panels.
Tag Static non-interactive label for metadata or categories.
Text Input Single-line field for entering freeform text.
Toast Transient notification that appears and auto-dismisses.
Toggle Switch On/off control for binary settings.
Tooltip Contextual popup that appears on hover or focus.
Video Embedded media player with accessible controls.
